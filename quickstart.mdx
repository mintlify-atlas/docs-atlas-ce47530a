## Prerequisites

Before you start, make sure you have the following:

- A modern browser with microphone access (Chrome or Firefox recommended)
- [Node.js](https://nodejs.org) 18+ and Yarn or npm
- Python 3.9+ with pip (for the Whisper backend)
- Three API keys:

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_OPENAI_API_KEY` | OpenAI API key for GPT-3.5-turbo evaluation |
| `NEXT_PUBLIC_ELEVEN_API_KEY` | ElevenLabs API key for text-to-speech feedback |
| `NEXT_PUBLIC_API_URL` | URL of your running Whisper Flask backend |

<Info>
  You can get an OpenAI API key at [platform.openai.com](https://platform.openai.com) and an ElevenLabs key at [elevenlabs.io](https://elevenlabs.io).
</Info>

## Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ehcaw/sfhacks.git
    cd sfhacks
    ```
  </Step>

  <Step title="Set up environment variables">
    Create a `.env.local` file in the project root:

    ```bash .env.local
    NEXT_PUBLIC_OPENAI_API_KEY=sk-...
    NEXT_PUBLIC_ELEVEN_API_KEY=xi-...
    NEXT_PUBLIC_API_URL=http://localhost:5000
    ```

    <Warning>
      All three variables are required. The app will not transcribe or evaluate without them. Because these variables use the `NEXT_PUBLIC_` prefix, they are exposed to the browser — do not commit `.env.local` to version control.
    </Warning>
  </Step>

  <Step title="Install frontend dependencies">
    <CodeGroup>

    ```bash yarn
    yarn install
    ```

    ```bash npm
    npm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Start the Whisper backend">
    The Whisper Flask server handles audio transcription. Install its dependencies and start it:

    ```bash
    pip install flask flask-cors openai-whisper torch
    python src/python/app.py
    ```

    The server starts on `http://localhost:5000` by default. It exposes a single endpoint — `POST /whisper` — that accepts an audio file and returns a transcript.

    <Note>
      If you have an NVIDIA GPU, Whisper automatically runs on CUDA for faster transcription. On CPU it still works, just more slowly.
    </Note>
  </Step>

  <Step title="Start the dev server">
    <CodeGroup>

    ```bash yarn
    yarn dev
    ```

    ```bash npm
    npm run dev
    ```

    </CodeGroup>

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>

  <Step title="Pick a topic">
    The homepage displays a bento grid of DSA topics:

    - Array
    - Linked list
    - Tree
    - Graph
    - Hash table
    - Heap
    - Sorting algorithms
    - Searching algorithms
    - FIFO / LIFO data structures

    Click any card to open the practice modal for that topic.
  </Step>

  <Step title="Record your explanation">
    When the modal opens, you'll see the prompt **"Explain to me what [topic] is"** along with a live audio visualizer.

    1. Click the microphone icon to start recording.
    2. Speak your explanation clearly — describe what the data structure is, how it works, and when you'd use it.
    3. Click the microphone icon again to stop recording.

    The `Recorder` component captures your audio as a WebM blob and immediately sends it to the Whisper backend for transcription:

    ```typescript src/components/ModulePopup.tsx
    const uploadAudio = async (blob: Blob) => {
      setIsLoading(true);
      const formData = new FormData();
      formData.append("audio", blob, "audio.webm");

      const response = await fetch(
        `${process.env.NEXT_PUBLIC_API_URL}/whisper`,
        {
          method: "POST",
          body: formData,
        }
      );

      const result = await response.json();
      const text = result?.results?.[0]?.transcript ?? "Could not hear user";
      // ...
    };
    ```

    A spinner overlay appears while the pipeline runs.
  </Step>

  <Step title="Receive spoken AI feedback">
    Once your audio is transcribed, it flows through two more steps automatically:

    1. **GPT-3.5-turbo** evaluates whether your explanation is correct and complete (up to 50 tokens of feedback).
    2. **ElevenLabs TTS** converts that text feedback into speech and plays it back through the hidden `<audio>` element on the modal.

    You'll hear the AI speak its assessment of your explanation directly in the browser.

    <Tip>
      If you don't hear anything, check that your browser hasn't blocked autoplay audio and that your `NEXT_PUBLIC_ELEVEN_API_KEY` is valid.
    </Tip>
  </Step>
</Steps>
