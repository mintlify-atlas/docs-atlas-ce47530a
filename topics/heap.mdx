title: Heap
description: A binary tree with a specific ordering property — every parent is smaller (min-heap) or larger (max-heap) than its children. Enables O(log n) insert and extract with O(1) peek.
---

A heap is a **complete binary tree** (all levels are full except possibly the last, which is filled from left to right) that satisfies the **heap property**:

- **Min-heap**: Every node's value is less than or equal to both of its children's values. The minimum element is always at the root.
- **Max-heap**: Every node's value is greater than or equal to both of its children's values. The maximum element is always at the root.

The source code in elitecode describes a heap as a "binary tree with a specific ordering property."

<Note>
  Heaps are almost always implemented with an **array**, not explicit tree nodes with pointers. The tree structure is implicit: for a node at index `i`, its left child is at `2i + 1`, its right child is at `2i + 2`, and its parent is at `Math.floor((i - 1) / 2)`.
</Note>

## Array representation

Given the array `[1, 3, 5, 7, 9, 8]`:

```
         1          ← index 0
       /   \
      3     5       ← indices 1, 2
     / \   /
    7   9 8         ← indices 3, 4, 5
```

Index relationships:
- Node at `i` → Left child at `2i + 1`, Right child at `2i + 2`
- Node at `i` → Parent at `Math.floor((i - 1) / 2)`

## How insertion and extraction work

**Insert** (`sift up`):
1. Append the new element at the end of the array (last position in the tree).
2. Compare it with its parent. If it violates the heap property, swap them.
3. Repeat step 2 until the heap property is restored or the element reaches the root.
4. Time: O(log n) — at most, the element travels from a leaf to the root.

**Extract min/max** (`sift down`):
1. Save the root value (that's your answer).
2. Move the last element in the array to the root position. Remove the last element.
3. Compare the new root with its children. If it violates the heap property, swap it with the smaller (min-heap) or larger (max-heap) child.
4. Repeat step 3 until the heap property is restored or the element reaches a leaf.
5. Time: O(log n) — at most, the element travels from the root to a leaf.

## Time and space complexity

| Operation     | Time     | Notes                                        |
|---------------|----------|----------------------------------------------|
| Peek min/max  | O(1)     | Root is always at index 0                    |
| Insert        | O(log n) | Sift up from leaf to root                    |
| Extract min/max | O(log n) | Sift down from root to leaf                |
| Build heap    | O(n)     | Floyd's algorithm: start from last internal node |
| Search        | O(n)     | No ordering for arbitrary elements           |

**Space complexity**: O(n) for n elements.

<Tip>
  Building a heap from an existing array with Floyd's algorithm is O(n) — not O(n log n). This is a common interview misconception. The proof involves summing work at each level of the tree.
</Tip>

## TypeScript code examples

```typescript
// Min-heap implementation backed by an array
class MinHeap {
  private heap: number[] = [];

  private parent(i: number): number {
    return Math.floor((i - 1) / 2);
  }
  private leftChild(i: number): number {
    return 2 * i + 1;
  }
  private rightChild(i: number): number {
    return 2 * i + 2;
  }
  private swap(i: number, j: number): void {
    [this.heap[i], this.heap[j]] = [this.heap[j], this.heap[i]];
  }

  // Insert — O(log n)
  insert(value: number): void {
    this.heap.push(value);
    this.siftUp(this.heap.length - 1);
  }

  private siftUp(i: number): void {
    while (i > 0 && this.heap[this.parent(i)] > this.heap[i]) {
      this.swap(i, this.parent(i));
      i = this.parent(i);
    }
  }

  // Extract minimum — O(log n)
  extractMin(): number | undefined {
    if (this.heap.length === 0) return undefined;
    if (this.heap.length === 1) return this.heap.pop();

    const min = this.heap[0];
    this.heap[0] = this.heap.pop()!; // move last to root
    this.siftDown(0);
    return min;
  }

  private siftDown(i: number): void {
    const n = this.heap.length;
    let smallest = i;
    const left = this.leftChild(i);
    const right = this.rightChild(i);

    if (left < n && this.heap[left] < this.heap[smallest]) smallest = left;
    if (right < n && this.heap[right] < this.heap[smallest]) smallest = right;

    if (smallest !== i) {
      this.swap(i, smallest);
      this.siftDown(smallest);
    }
  }

  // Peek — O(1)
  peek(): number | undefined {
    return this.heap[0];
  }

  size(): number {
    return this.heap.length;
  }
}

// Find K largest elements — O(n log k)
function kLargest(nums: number[], k: number): number[] {
  const minHeap = new MinHeap();
  for (const num of nums) {
    minHeap.insert(num);
    if (minHeap.size() > k) {
      minHeap.extractMin(); // evict the smallest so far
    }
  }
  // What remains in the heap are the k largest elements
  const result: number[] = [];
  while (minHeap.size() > 0) result.push(minHeap.extractMin()!);
  return result;
}
```

<Tip>
  To find the **K smallest** elements, use a **max-heap** of size K — evict whenever you exceed K elements. To find the **K largest** elements, use a **min-heap** of size K. This counterintuitive pairing trips up many candidates.
</Tip>

## Common use cases

- **Priority queues**: Operating systems schedule processes by priority; event simulators process events in time order. A heap is the canonical priority queue implementation.
- **Heap sort**: Build a max-heap, then repeatedly extract the max to produce a sorted array. O(n log n) time, O(1) space (in-place).
- **Finding kth largest/smallest**: Maintain a heap of size K for an online (streaming) solution.
- **Dijkstra's shortest path**: A min-heap on (distance, vertex) enables O((V + E) log V) shortest path.
- **Median of a stream**: Maintain two heaps — a max-heap for the lower half and a min-heap for the upper half — rebalancing as needed.

## How to explain it in elitecode

**Start with the definition:**
> "A heap is a complete binary tree with the heap property. In a min-heap, every parent is smaller than its children, so the minimum is always at the root. In a max-heap, every parent is larger, so the maximum is always at the root."

**Explain the array representation:**
> "Heaps are stored as arrays, not as linked nodes. The root is at index 0. For any node at index i, its left child is at 2i + 1 and its right child is at 2i + 2. Its parent is at floor((i-1)/2). This lets you navigate the tree with just arithmetic — no pointers needed."

**Walk through insert and extract:**
> "To insert, I append to the end of the array and sift up — compare with the parent and swap if the heap property is violated. Keep going until it's in the right place. That's at most log n swaps. To extract the min, I take the root, move the last element to the root position, then sift down — swap with the smaller child until the property is restored. Also log n swaps."

**Close with the top-K pattern:**
> "The most common interview application is finding the K largest elements. I use a min-heap of size K. For each element: if the heap has fewer than K elements, push it. If the new element is larger than the heap's minimum, pop the min and push the new element. The min-heap acts as a filter — it keeps the K largest seen so far. O(n log K) total."

<Warning>
  Don't confuse the heap data structure with "the heap" in memory management (where dynamically allocated objects live). They share a name but are entirely different concepts. Be explicit in your explanation.
</Warning>

## Common interview questions

<AccordionGroup>
  <Accordion title="What is the difference between a min-heap and a max-heap?">
    In a **min-heap**, every parent node is less than or equal to its children, so the root holds the minimum element. In a **max-heap**, every parent is greater than or equal to its children, so the root holds the maximum. Both support O(log n) insert and extract, and O(1) peek. The only difference is the comparison direction used in sift-up and sift-down.
  </Accordion>
  <Accordion title="Why is building a heap from an array O(n) and not O(n log n)?">
    Inserting n elements one at a time is O(n log n). But Floyd's algorithm builds the heap in-place by starting from the last internal node (index `n/2 - 1`) and calling sift-down on each node going toward the root. Nodes near the leaves do very little work (sift-down of O(1)); only the root sifts down O(log n). Summing the work across all levels gives a geometric series that converges to O(n).
  </Accordion>
  <Accordion title="How do you find the median of a data stream?">
    Maintain two heaps: a **max-heap** for the lower half of numbers and a **min-heap** for the upper half. Invariant: the max-heap's size is equal to or one greater than the min-heap's size. The median is the max-heap's root (odd total) or the average of both roots (even total). On each insert: push to the appropriate heap, then rebalance if sizes differ by more than 1. Each operation is O(log n).
  </Accordion>
  <Accordion title="What is heap sort and how does it work?">
    Heap sort is an in-place O(n log n) comparison sort. Step 1 (build max-heap): apply Floyd's algorithm to turn the input array into a max-heap — O(n). Step 2 (extract): repeatedly swap the root (max) with the last element, shrink the heap size by 1, and sift down the new root — O(n log n). After all extractions, the array is sorted in ascending order. It is not stable (equal elements may change relative order), but it uses O(1) extra space.
  </Accordion>
  <Accordion title="How would you find the kth largest element efficiently?">
    **Option 1 — min-heap of size k (online)**: Iterate through all n elements. Maintain a min-heap of size k. For each element, push it; if size exceeds k, extract the min. At the end, the root is the kth largest. Time: O(n log k), Space: O(k).

    **Option 2 — QuickSelect (offline)**: Use the partition step from quicksort. Average O(n) time, O(1) extra space, but O(n²) worst case. The heap approach is preferred when worst-case guarantees matter or the input is a stream.
  </Accordion>
</AccordionGroup>
